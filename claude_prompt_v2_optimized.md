# 热点运营Agent平台 - 全面重构提示词 V2（优化版）

## 一、产品定位与核心设计哲学

我们要做一个**AI Agent驱动的热点运营平台**，服务于抖音内容运营团队。核心链路：热点线索感知 → 事件挖掘 → 运营执行 → 效果复盘。

**不是传统ERP后台，而是"运营的AI副驾驶"**。当前demo犯了一个根本性错误——用传统OA系统的思维做了一个AI Agent产品，界面充斥着静态卡片、平铺标签、纯文本块，完全没有"智能感"。

**五个不可妥协的设计原则**：
1. 每个AI判断必须有"为什么"——必须展示推理链路
2. 每个建议必须有来源——SOP、历史案例、数据依据必须可见可查
3. Agent协作必须可视化——多Agent的分工和传递必须让用户感知到
4. 决策必须有支撑——确认前必须看到效果预估、风险对比、相似案例
5. 复盘必须能闭环——复盘→调优→A/B验证→知识库更新的链路完整可见

## 二、系统架构（必须在UI中体现）

### 2.1 多Agent架构：分层集中式

**不是四个平级Agent流水线，而是Orchestrator + 领域Agent的分层架构。**

```
Orchestrator（中央调度Agent）
├─ 感知Agent：信号采集、去噪、初筛
├─ 挖掘Agent：事件聚类、实体抽取、生命周期判定
├─ 运营Agent：SOP匹配、策略生成、动作编排
└─ 复盘Agent：效果归因、策略评价、知识沉淀
```

**Orchestrator的核心职责**（必须在UI中体现）：
- 全局状态机管理（每个事件的状态流转）
- 动态任务编排（不是固定A→B→C，支持跳过/回退/分支）
- 人机卡点决策（关键节点暂停等待人工确认）
- 异常兜底（超时/失败时重试/降级）

**在UI中的体现方式**：
- Tab 2（事件详情）右栏必须展示Agent推理链路可视化（三节点横向排列+箭头连接+脉冲动画）
- Tab 3（Agent编排）必须展示流程流水线可视化（水平节点+连线流动动画）
- 全局必须有Agent状态指示器（顶部Agent工作状态区域）

### 2.2 RAG架构：Hybrid（语义层+Skill + VectorRAG + GraphRAG）

**不是替代关系，是互补——结构化查询走语义层+Skill，开放理解走RAG。**

| 路由目标 | 适用场景 | UI中如何体现 |
|---------|---------|-------------|
| 语义层+Skill | 热度指标计算、SOP匹配、归因分析 | 显示"调用EventScoring Skill"标签，展示预定义指标拆解 |
| VectorRAG | 相似历史事件检索、开放域内容理解 | 显示"基于知识图谱+向量召回"标签，展示相似度百分比 |
| GraphRAG | 事件关联分析、多跳实体推理 | 显示实体关系图谱或关联事件链 |

### 2.3 语义层+Skill设计（必须在运营推荐中体现）

热度由6个原子指标加权计算，每个因子必须可视化：
- 时效信号（满分28）、传播速度（满分25）、作者密度（满分15）
- 情感倾向（满分15）、内容质量（满分15）、平台适配度（满分17）
- 总分 = Σ(wi × 原子指标i)

Skill库：EventScoring、LifecyclePredict、SOPMatch、Attribution、AnomalyDetect、Forecast、ReportGenerate。每个Skill调用时在UI中显示调用标签。
